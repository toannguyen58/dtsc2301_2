Blackjack Reinforcement Learning — Q-Learning + Decision Tree + Logistic Regression

April 2026

Full project code, notebooks, and data are available on GitHub: github.com/toannguyen58

1. Project Overview

This project builds a blackjack AI in three stages:

  1. Reinforcement learning — a Q-learning agent learns to play blackjack from scratch using trial and error across 500,000 hands.
  2. Logging — every hand is recorded to disk, capturing the game state, the actions taken, and the outcome.
  3. Supervised modeling — the logged data is used to train a decision tree and a logistic regression classifier to predict hand outcomes, with the Q-table's learned values encoded as features.

The environment is a custom 8-deck shoe with four player actions: stand, hit, double-down, and split.

Why this project? In blackjack, players face a long-term disadvantage — winning only about 42% of hands, while the dealer wins close to 49%, with the rest resulting in ties. These outcomes are driven by the probabilistic structure of the game under consistent play. This raises a key question: can an agent, learning purely through trial and error, discover a strategy that improves upon these baseline outcomes?

2. Environment Design

The Shoe

Eight standard 52-card decks are shuffled together into a 416-card shoe. When fewer than 104 cards remain (~25%), the shoe reshuffles. This simulates real casino conditions where the shoe is never fully depleted before reshuffling.

State Representation

At any point in a hand, the agent observes a 4-tuple:

state = (player_total, dealer_up_card, usable_ace, can_split)
ComponentDescription
player_totalOptimal sum of the player's hand (aces counted as 11 if it doesn't bust)
dealer_up_cardNumeric value of the dealer's one face-up card (2–10, A=1)
usable_aceTrue if the player holds an ace currently counted as 11 (soft hand)
can_splitTrue if this is the first action and both starting cards share the same point value

This is a discrete, finite state space. After 500,000 episodes the agent had encountered exactly 360 unique states.

Actions

CodeNameRuleReward
0StandStop; dealer plays out±1 or 0
1HitDraw one card±1 on terminal (0 if continuing)
2Double-downFirst action only — draw exactly one card, then stand±2 (doubled stake)
3SplitFirst action only, same-value starting cards — split into two handsSum of both outcomes (−2 to +2)

Double-down and split are only offered when they are legally available. The environment exposes env.valid_actions() at every step, and the agent's action selection is restricted to that list.

Dealer Rule

The dealer hits until reaching a hard or soft total of 17 or more, then stands (standard casino S17 rule).

3. Q-Learning Agent

What Q-Learning Does

Q-learning is a model-free reinforcement learning algorithm. The agent does not know the rules of blackjack in advance — it discovers a strategy purely through experience. After taking an action, it observes the reward and updates its internal table of expected values using the Bellman equation: 

Q(s, a)  ←  Q(s, a)  +  α · [ r  +  γ · max Q(s')  −  Q(s, a) ]

Each term has a specific role:

  • Q(s, a) — current estimate of how good action a is in state s.
  • r — immediate reward received after taking the action.
  • γ · max Q(s') — discounted value of the best possible action from the next state (0 for terminal states).
  • α — controls how aggressively the estimate is updated.

The Q-Table

The Q-table is a dictionary:

Q[(player_total, dealer_up, usable_ace, can_split)] = [stand_value, hit_value, double_value, split_value]

Optimistic initialization: every state starts at [1.0, 1.0, 1.0, 1.0]. This forces the agent to try every action in every state at least once before settling — because an untried action looks better than a tried-and-failed one.

Exploration vs Exploitation (Epsilon-Greedy)

Epsilon (ε) controls the exploration/exploitation tradeoff. With probability ε the agent takes a random valid action (explore); otherwise it picks the highest-value valid action (exploit).

Epsilon decays exponentially toward a floor of 0.1:

ε  =  ε_min  +  (ε_start − ε_min) · e^(−decay_rate × episode)

With decay_rate = 0.0001 over 500,000 episodes, epsilon reaches its floor after roughly episode 428, giving the agent ~499,572 episodes of near-greedy exploitation. The floor of 0.1 ensures the agent never completely stops exploring.

6. Supervised Models

6.1 Decision Tree

A DecisionTreeClassifier is trained to predict hand outcome from the feature set above. The tree is interpretable — you can trace exactly why a hand was classified WIN or LOSE — and handles the mix of continuous Q-values and discrete game state naturally without scaling.

Depth tuning via 5-fold cross-validation (depths 1–20):

Best depth: 7 (test accuracy 64.64% vs baseline depth=5 at 64.53%). The gain is marginal — the CV curve was flat across depths 5–15, confirming the predictive signal is exhausted after just a few splits.

player_total is the dominant feature. Q-table values (stand_value, hit_value, etc.) contribute meaningfully at deeper splits. See the feature importance chart in the notebook for exact percentages.

6.2 Random Forest

A RandomForestClassifier (100 estimators) trained on the same feature set. Ensembles many shallow trees to reduce variance. Interestingly, it is the only model that predicts PUSH at all (9% recall, F1=0.13).

6.3 Logistic Regression

A linear baseline trained after standardizing all features with StandardScaler. Outputs calibrated probabilities. If the tree significantly outperforms it, that confirms the decision boundary is non-linear.

Key Observations

  1. PUSH is essentially unpredictable. All models give up on predicting PUSH (F1 ≈ 0.00). A push requires the player's final total to exactly equal the dealer's final total — an outcome that depends on the dealer's hidden card and future draws, information not in the feature set.
  2. The tree predicts LOSE with higher recall. 77% recall on LOSE vs 70% for logistic regression. Threshold-based splits like player_total < 12 capture near-certain losing hands cleanly.
  3. Logistic regression handles WIN better. 74% recall on WIN — the decision boundary for winning hands is more linearly separable (high player total + weak dealer card = WIN).
  4. The accuracy ceiling is around 64–65%. The remaining ~35% loss comes from unpredictable pushes and inherent outcome randomness — two identical visible states can have different outcomes depending on the dealer's hidden card.
  5. Random Forest underperforms but recovers partial PUSH signal. At 61.2%, it trails the single tree and logistic regression. However, it is the only model that predicts PUSH at all (9% recall, F1=0.13) — ensemble averaging picks up some minority-class signal that single-threshold models miss.
  6. CV tuning had almost no effect. Depth=7 vs depth=5: 64.64% vs 64.53%. The CV curve was flat between depths 5–15.

8. Tree vs Q-Table Action Comparison

A separate decision tree was trained to predict the Q-table's recommended action (hit / stand / double / split) from just the three state features (player_total, dealer_up, usable_ace).

Agreement with Q-table: 100%

The tree perfectly replicates the Q-table policy. This is expected: the Q-table is itself a mapping from the same features to an action — a decision tree over the same inputs can perfectly reproduce any lookup table given enough depth.

The policy heatmap (player_total × dealer_up) shows the learned strategy: always hit below ~12 regardless of dealer card, stand on strong totals (18+), and the 13–17 range is the contested zone where dealer card matters.

9. Limitations

Split Action (Fixed)

Adding can_split as a 4th state dimension fixed a prior artifact where split was over-recommended. After retraining, split is recommended in only 2 of 360 states (0.6%), down from 173 states in earlier runs.

PUSH Is Not Predictable

The supervised models cannot reliably predict pushes. This is a structural limitation — the dealer's hidden card and future draws are unknown at decision time. A model would need the dealer's full hand to predict pushes, which defeats the purpose of prediction.

Reward Scale Inconsistency

Double-down uses ±2 rewards while stand and hit use ±1. The Q-table's double_value entries live on a different scale. On a normalized basis (÷2), the double mean is approximately −0.25, consistent with the other actions. This also inflates the negative mean_double_value statistic.

Training Volume for New Actions

500,000 episodes is sufficient for stand and hit to converge (available every hand). Double is available on every hand's first action, split only when starting cards match — so their Q-values receive fewer updates and converge more slowly. A longer training run would give more reliable double/split recommendations.
